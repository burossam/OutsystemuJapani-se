Web アプリ作成

課題説明

この演習は、OutSystems で Web アプリを段階的に開発していくための手引きとなる演習シリーズの初回である。

作成する映画アプリの機能について
目的 映画とそれに携わった人物（キャストおよびスタッフ）のDB作成し、維持すること
映画情報 タイトル、あらすじ、ジャンルなど
人物 名前や生年月日のほか、携わった映画やそこでの役割などの略歴
映画アプリユーザー こうした全情報にアクセスできるだけでなく、映画についてのコメントや評価も行う。
こうした評価は、他ユーザーが参照して映画に関する平均的評価や全コメント確認できるように集約する。
映画アプリの管理者 映画の新規追加や既存映画を編集が可能。
俳優やスタッフを追加・編集して、映画に関連付けも可能。

※この先、より高度な新しい OutSystems の概念について学習を進めながら、下記３つを継続的に行う。

  • アプリ拡張
  • パブリッシュ
  • テスト

この演習を修了する頃には、小規模ではあるものの、複数画面と概念を利用した、ブラウザから簡単にアクセスできるアプリが完成しているはず

2.3xで行う演習内容

この演習では主に下記２つのモジュールを含むWEBアプリを作成する。

モジュール名 役割
Web Responsive UI とビジネスロジックをすべて保持
Blank Core データモデルを保持

シナリオ作成

アプリテストするために、ユーザーがブラウザでアプリを開くと「“HelloWorld”」というメッセージが表示

２つの目的

モジュール 役割
Core モジュールの要素参照 実現する為にはアプリ名を返すパブリックアクションを Core モジュールに作成が必要
UI モジュールの画面表示 ユーザーがアプリを開くとアクション結果がブラウザに表示される。
